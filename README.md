# Интеграция Росдомофон для Home Assistant

Интеграция позволяет управлять устройствами Росдомофон (двери подъезда, шлагбаумы, ворота, калитки), а также просматривать камеры видеонаблюдения из Home Assistant через облачный API Росдомофон.

## Возможности

### 🔓 Замки и доступ
- **Управление замками** — замки Росдомофон как сущности `lock` в Home Assistant.
- **Разные типы устройств** — дверь подъезда, шлагбаум, ворота, калитка (отдельные сущности с иконками).
- **Гостевые ссылки** — временная ссылка для открытия двери гостю без доступа к Home Assistant (настраиваемый TTL, автоотзыв).

### 📹 Камеры
- **Просмотр камер** — HLS-потоки камер Росдомофон прямо в интерфейсе HA.
- **Прокси с авторизацией** — потоки проксируются с bearer-токеном и собственной подписью, работают и локально, и удалённо.

### 👤 Распознавание лиц (авто-открытие по лицу)
- **Авто-открытие двери** по «своему» лицу через self-hosted **DeepFace** (с антиспуфингом).
- **Предфильтр по движению** — DeepFace не вызывается на статичных кадрах, детекция лица вынесена в DeepFace (без тяжёлых зависимостей).
- **Лента активности** — кадры распознанных и незнакомых лиц в **Логбуке** + image-сущности с последними кадрами.
- **Добавление лиц по ссылке** — временная ссылка с предпросмотром камеры: снять лицо с камеры или загрузить фото, **авто-обрезка** лица, **несколько фото на человека**, удаление по одному.
- **Галерея лиц** — страница со всеми людьми и эскизами их фото: добавление фото (загрузкой), удаление отдельного фото или человека целиком (из меню «Люди» в настройках).
- **Отладочная галерея** — по включению отладки: страница с кадрами, отправленными в DeepFace, и результатом по каждому (для подбора порога/детектора).

## Требования

- Home Assistant **2023.9+**.
- Аккаунт в сервисе **Росдомофон** с привязанным номером телефона РФ.
- Установленный **HACS** (Home Assistant Community Store).

## Установка через HACS

1. Откройте **HACS → Integrations** в интерфейсе Home Assistant.
2. Нажмите кнопку **⋮ (три точки)** в правом верхнем углу и выберите **Custom repositories**.
3. В поле **Repository** укажите URL репозитория:
   - `https://github.com/iljyxa/rosdomofon-ha`
4. В поле **Category** выберите **Integration** и нажмите **Add**.
5. Найдите интеграцию **Росдомофон** в списке HACS.
6. Нажмите **Install** и перезагрузите Home Assistant.

## Настройка интеграции

1. Перейдите в **Настройки → Устройства и службы → Добавить интеграцию**.
2. Найдите **Росдомофон** (или `rosdomofon`).
3. Введите номер телефона РФ (можно в любом формате: `+7 (999) 123-45-67`, `89991234567` и т.д.).
4. Введите код из SMS.
5. После успешной авторизации будут созданы сущности замков и кнопки «Поделиться» для каждого замка.

## Гостевые ссылки (Поделиться)

Интеграция позволяет создавать временные ссылки для открытия двери гостями без доступа к Home Assistant.

### Кнопка «Поделиться»

Для каждого замка автоматически создаётся кнопка `button.rosdomofon_share_<id>`. При нажатии:

1. Генерируется уникальная ссылка со сроком действия **12 часов**.
2. Ссылка отображается в уведомлении Home Assistant — скопируйте её и отправьте гостю.
3. Гость переходит по ссылке в браузере — открывается страница с кнопкой «ОТКРЫТЬ» и информацией о том, сколько ещё действует ключ.
4. При нажатии на кнопку на странице выполняется попытка открыть замок; при успехе гость видит статус «ОТКРЫТО», при ошибке — «ОШИБКА» и текст ошибки.
5. По истечении срока ссылка автоматически деактивируется.

### Сервис `rosdomofon.generate_share_link`

Альтернативно можно вызвать сервис из автоматизаций или инструментов разработчика:

```yaml
service: rosdomofon.generate_share_link

  entity_id: lock.rosdomofon_12345_1
  ttl_hours: 12
```

Параметры:

| Параметр    | Описание                               | По умолчанию |
|-------------|----------------------------------------|--------------|
| `entity_id` | Entity ID замка Росдомофон             | Обязательный |
| `ttl_hours` | Срок действия ссылки в часах (0.5–168) | 12           |

### Требования для гостевых ссылок

- Должен быть настроен **внешний доступ** к Home Assistant (External URL или Nabu Casa).
- Если внешний доступ не настроен, при нажатии кнопки появится уведомление с инструкцией.

### Безопасность

- Каждая ссылка содержит уникальный UUID и не требует авторизации в HA.
- Ссылка автоматически деактивируется по истечении срока.
- При выгрузке / удалении интеграции все активные ссылки отзываются.
- Рекомендуется использовать HTTPS для внешнего доступа.

## Камеры видеонаблюдения

Интеграция автоматически обнаруживает камеры, привязанные к вашему аккаунту Росдомофон, и создаёт для них сущности типа `camera`.

### Возможности камер

- Просмотр живого HLS потока в интерфейсе Home Assistant
- Автоматическая авторизация запросов через встроенный прокси-сервер
- Поддержка просмотра через веб-интерфейс и мобильные приложения Home Assistant
- Интеграция с картами (Lovelace) и панелями мониторинга

### Технические требования

#### Для локального просмотра
- Браузер с поддержкой HLS (Safari, Edge) или расширение для Chrome/Firefox
- Home Assistant должен быть доступен по HTTPS (для корректной работы медиа-потоков)

#### Для удалённого просмотра
- Настроенный **внешний доступ** (External URL) или **Home Assistant Cloud** (Nabu Casa)
- HTTPS соединение (обязательно для безопасной передачи видео)

### Как работает прокси

Для просмотра потоков с камер Росдомофон требуется авторизация по Bearer токену. Интеграция автоматически:

1. Перехватывает запросы к HLS потокам камер
2. Добавляет заголовок `Authorization` с актуальным токеном
3. Переписывает URL в плейлистах для корректной работы
4. Возвращает поток клиенту

Дополнительно потоковый URL в Home Assistant подписывается (signed URL), чтобы стрим-воркер мог получать доступ без логина в UI. На старых версиях HA, где подпись недоступна, используется обычный (неподписанный) URL.

Всё это происходит прозрачно — вам не нужно ничего настраивать вручную.

### Использование в картах Lovelace

```yaml
type: picture-glance
camera_image: camera.rosdomofon_camera_39167
entities:
  - lock.rosdomofon_entrance_door
  - button.rosdomofon_share_entrance_door
```

Или для просмотра потока:

```yaml
type: custom:webrtc-camera
entity: camera.rosdomofon_camera_39167
```

## Распознавание лиц (авто-открытие двери)

Интеграция умеет автоматически открывать замок, когда камера видит «своё» лицо. Распознавание выполняет отдельный сервис **DeepFace** (свободный, self-hosted), который также проверяет кадр на подделку (антиспуфинг: печатное фото, экран телефона, маска).

### 1. Запуск сервиса DeepFace

Поднимите контейнер DeepFace на вашем сервере:

```bash
docker run -d --name deepface --restart unless-stopped \
  -p 5000:5000 serengil/deepface
```

Сервис будет доступен по адресу `http://<адрес-сервера>:5000`. При первом распознавании DeepFace скачает модели (нужен доступ в интернет).

### 2. Настройка в Home Assistant

Перейдите в **Настройки → Устройства и службы → Росдомофон → Настроить**. Откроется меню:

- **Настройки сервиса** — укажите URL DeepFace (`http://<адрес-сервера>:5000`), модель (по умолчанию `Facenet512`), детектор лиц (по умолчанию `opencv` — быстрый и без загрузки моделей; `retinaface`/`mtcnn` точнее, но тяжелее и скачивают доп. модели), порог сходства (меньше — строже), интервал опроса, паузу после открытия, антиспуфинг, **предфильтр кадров** (по умолчанию включён) и **отладку** (галерея кадров, отправленных в DeepFace; по умолчанию выключена).
- **Люди** — открывает **галерею лиц**: страница со всеми людьми, где можно добавить фото (загрузкой), посмотреть эскизы и удалить отдельные фото или человека целиком. Фото автоматически обрезается по лицу перед сохранением; на человека можно несколько фото. (Съёмка лица прямо с камеры — через ссылку `generate_enroll_link`.)
- **Камеры и замки** — для каждой камеры выберите замок, который открывать при распознавании. Пусто — авто-открытие для камеры выключено.

### 3. Как работает

Раз в N секунд интеграция берёт кадр с включённой камеры. Перед тяжёлым распознаванием работает дешёвый **предфильтр по движению**: кадр сравнивается с предыдущим, и если сцена статична — DeepFace не вызывается вовсе (домофон почти всегда смотрит в пустой двор). Если движение есть — кадр уходит в DeepFace.

Проверка «есть ли лицо в кадре» выполняется самим DeepFace (`enforce_detection`): если лицо не найдено, кадр отбрасывается и не тратит распознавание — так пустые кадры не попадают в ленту как «неизвестное лицо». При найденном лице DeepFace проверяет на подделку и сравнивает с эталонами; при совпадении выше порога открывает замок, шлёт уведомление и обновляет сенсор.

Предфильтр по движению можно отключить в настройках сервиса (тогда каждый кадр по интервалу уходит в DeepFace). Отдельный локальный детектор лиц не требуется: на свежих сборках Home Assistant (Python 3.14) библиотека OpenCV не устанавливается, поэтому детекция лиц вынесена в контейнер DeepFace, где OpenCV уже есть.

### Лента активности лиц

Каждое обнаруженное лицо попадает в **Логбук** (ленту активности) — и распознанное, и незнакомое:

- В Логбуке появляются строки вида «Росдомофон: распознан(а) Иван (camera.dvor)» и «Росдомофон: обнаружено неизвестное лицо (camera.podezd)». Каждая строка привязана к сущности с кадром — по клику открывается сам снимок.
- Незнакомые лица записываются с паузой (кулдаун камеры), чтобы одно и то же лицо не засоряло ленту на каждом кадре. Дверь при этом не открывается.
- Кадры хранятся в памяти и сбрасываются при перезапуске Home Assistant (как превью камер).

### Сущности

- `switch.rosdomofon_face_unlock_<камера>` — включить/выключить авто-открытие на камере.
- `sensor.rosdomofon_last_recognized_face` — последнее распознанное лицо (для автоматизаций).
- `image.rosdomofon_last_recognized_face` — кадр последнего распознанного лица.
- `image.rosdomofon_last_unknown_face` — кадр последнего нераспознанного лица.

### Отладка распознавания

Если распознавание работает не так, как ожидается (не открывает / открывает не тем / срабатывает предфильтр), включите **Отладку** в настройках сервиса. Тогда каждый кадр, отправленный в DeepFace, попадает в галерею с результатом:

- Открывается по адресу `/api/rosdomofon/debug` — ссылку с подписью Home Assistant покажет уведомление сразу после включения отладки.
- По каждому кадру видно: время, камера, время обработки, сколько лиц найдено, совпадение и расстояние (а при отсутствии совпадения — ближайший эталон и его расстояние, чтобы подобрать порог), либо подделка/ошибка DeepFace.
- Хранятся последние 25 кадров в памяти (сбрасываются при перезапуске). В обычном режиме, без отладки, ничего не собирается и ресурсы не тратятся.

### Добавление лиц по ссылке

Чтобы не загружать фото человека вручную, можно дать ему **ссылку для самостоятельного добавления лица** (аналог гостевой ссылки, но для регистрации лица).

- Сервис **`rosdomofon.generate_enroll_link`** (Инструменты для разработчика → Действия, или из автоматизации): выбираешь **камеру** и **человека** (существующего или вводишь нового), опционально срок действия (по умолчанию 12 ч). Ссылка приходит в уведомлении.
- По ссылке открывается страница с **живым предпросмотром камеры**. На ней две кнопки: **«Снять с камеры»** (сервер берёт текущий кадр) и **«Загрузить фото»** (с телефона/галереи).
- Найденное лицо **автоматически обрезается** и добавляется в эталоны человека. Можно сделать **несколько фото** с разных ракурсов — все эскизы видны на странице, любой можно удалить.
- Ссылка действует ограниченное время (TTL) и защищена неугадываемым идентификатором.

> ⚠️ **Безопасность:** по этой ссылке можно добавить лицо в список «своих» — то есть фактически выдать доступ на открытие двери. Не пересылайте ссылку посторонним и держите TTL небольшим.

### ⚠️ Ограничения и безопасность

- Антиспуфинг Silent-Face защищает от печатных фото и экранов, но **не абсолютен** (качественные маски и др. могут пройти). Это удобство, а не банковский уровень доступа.
- Качество распознавания зависит от освещения и ракурса домофонной камеры — возможны пропуски. Настраивается порогом.
- Опрос камеры каждые несколько секунд держит поток и нагружает DeepFace (инференс на CPU). Увеличьте интервал, если нагрузка высокая.
- Эталонные эмбеддинги хранятся локально в Home Assistant; сами фото в облако не отправляются.

## Примеры автоматизаций

### Автоматическое создание гостевой ссылки по расписанию

```yaml
automation:
  - alias: "Гостевая ссылка для курьера"
    description: "Создаёт временную ссылку каждое утро для курьера"
    trigger:
      - platform: time
        at: "09:00:00"
    condition:
      - condition: time
        weekday:
          - mon
          - tue
          - wed
          - thu
          - fri
    action:
      - service: rosdomofon.generate_share_link
        
          entity_id: lock.rosdomofon_entrance_door
          ttl_hours: 2
      - service: notify.telegram
        
          message: "Ссылка для открытия двери создана и действует до 11:00"
```

## Отладка и диагностика

Интеграция использует логгер `custom_components.rosdomofon`. Для подробной диагностики:

```yaml
logger:
  default: warning
  logs:
    custom_components.rosdomofon: debug
```

## Безопасность

- Все запросы к API Росдомофон используют HTTPS
- Токены доступа хранятся в защищённом хранилище Home Assistant
- Гостевые ссылки содержат уникальный UUID и автоматически истекают
- Рекомендуется использовать HTTPS для внешнего доступа к Home Assistant
